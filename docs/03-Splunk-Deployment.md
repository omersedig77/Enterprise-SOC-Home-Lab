# Splunk Enterprise Deployment

## Overview

Splunk Enterprise serves as the Security Information and Event Management (SIEM) platform for this SOC Homelab. It is responsible for collecting, indexing, searching, and visualizing security telemetry generated by Windows, Linux, and network infrastructure.

The deployment was configured to centralize log collection from multiple data sources, enabling detection engineering, real-time alerting, dashboard development, and incident investigation.

---

# Environment

| Component | Value |
|-----------|-------|
| Operating System | Ubuntu Desktop 22.04 |
| Splunk Version | Splunk Enterprise 10.4.0 |
| Management Port | 8000 |
| Receiving Port | 9997 |

---

# Installation

Splunk Enterprise was installed using the official Debian package (.deb) provided by Splunk.

After installation, the Splunk service was configured to:

- Start automatically during system boot
- Accept the Splunk license agreement
- Create the administrator account
- Enable the Splunk Web interface

The web interface can be accessed using:

```
http://192.168.50.101:8000
```

or locally from the server:

```
http://127.0.0.1:8000
```

---

# Receiving Configuration

Splunk Enterprise was configured to receive data from Splunk Universal Forwarders using the default receiving port:

```
9997
```

Both monitored endpoints forward their logs directly to:

```
192.168.50.101:9997
```

This configuration enables centralized log collection from multiple operating systems.

---

# Index Design

To improve data organization and simplify searches, separate indexes were created for each major log source.

| Index | Purpose |
|--------|---------|
| wineventlog | Windows Event Logs |
| linux_logs | Linux System Logs |
| firewall | pfSense Firewall Logs |

Using dedicated indexes allows searches, dashboards, and alerts to target specific data sources while keeping the environment organized.

---

# Current Data Sources

## Windows Endpoint

The Windows endpoint forwards the following logs:

### Windows Event Logs

- Security
- System
- Application

### Sysmon

- Microsoft-Windows-Sysmon/Operational

These logs provide endpoint visibility for authentication events, process creation, service activity, PowerShell execution, network connections, registry modifications, and other host-based security events.

---

## Linux Endpoint

The Ubuntu endpoint forwards:

- /var/log/syslog
- /var/log/auth.log
- /var/log/kern.log

These logs provide visibility into authentication events, system activity, kernel events, and administrative actions.

---

## pfSense Firewall

The pfSense firewall forwards firewall logs into the dedicated **firewall** index.

Firewall telemetry provides visibility into:

- Allowed connections
- Blocked traffic
- Network activity
- Firewall events
